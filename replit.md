# NovaMind AI

A full-featured AI assistant app — think GPT + Siri + creative studio. Converses, generates images, humanizes text, writes code, composes songs, extracts YouTube transcripts, and exports conversations as PDF.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080)
- `pnpm --filter @workspace/ai-assistant run dev` — run the frontend (port 19570)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL` — Postgres connection string
- Required env: `AI_INTEGRATIONS_OPENAI_BASE_URL`, `AI_INTEGRATIONS_OPENAI_API_KEY` — Replit AI Integrations (auto-set)

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- AI: OpenAI via Replit AI Integrations (gpt-5.4, gpt-5.3-codex, gpt-image-1, gpt-audio)
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)
- Frontend: React + Vite + shadcn/ui + Tailwind + Framer Motion

## Where things live

- `lib/api-spec/openapi.yaml` — API contract (source of truth)
- `lib/db/src/schema/` — Drizzle ORM schema (conversations.ts, messages.ts)
- `artifacts/api-server/src/routes/` — Express route handlers
  - `openai/conversations.ts` — chat CRUD + SSE streaming
  - `openai/voice.ts` — voice chat (speech-to-speech)
  - `openai/image.ts` — image generation
  - `tools/index.ts` — humanizer, youtube, code, song, pdf export
- `artifacts/ai-assistant/src/` — React frontend
- `lib/integrations-openai-ai-server/` — OpenAI SDK wrapper (server)
- `lib/integrations-openai-ai-react/` — Voice hooks (client)

## Architecture decisions

- Contract-first API: all endpoints defined in OpenAPI spec, types generated by Orval
- SSE streaming for chat: `text/event-stream` for real-time token delivery
- Voice uses gpt-audio for speech-to-speech with the `voiceChatStream` helper
- Image generation uses gpt-image-1 with style-augmented prompts
- Code assistant uses gpt-5.3-codex (coding-optimized model)
- All conversations and messages persisted in PostgreSQL

## Product

- **Chat** — AI conversation with markdown, streaming responses, conversation history
- **Image Generator** — Text-to-image with 4 styles and 3 sizes
- **Text Humanizer** — Convert AI text to human-like writing
- **YouTube Transcript** — Extract + summarize video content
- **Code Assistant** — Generate, debug, explain code in multiple languages
- **Song Generator** — Write lyrics by mood, genre, and language
- **Voice** — Real-time voice conversation (click mic icon)
- **PDF Export** — Export any conversation

## Gotchas

- Always run `pnpm --filter @workspace/db run push` after schema changes
- Re-run codegen after every openapi.yaml change
- Audio payloads require large body limits (50mb set in app.ts)
- SSE endpoints (chat, voice) cannot use generated React Query hooks — use fetch + ReadableStream
